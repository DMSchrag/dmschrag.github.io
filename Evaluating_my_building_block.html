Mastering Tinkering

Evaluation of My Building Block

Looking back at my design and how it fits the properties of tinkering materials.

Design Evaluation

Affordability

One thing I noticed quite quickly is that cost plays a big role when building something like this. The parts I used are not extremely expensive, but once you start looking at better motors or higher quality components, the price goes up very fast.

Compared to something like a full professional setup, this is still relatively affordable, but it is definitely not “cheap”, especially if you want it to work reliably and not just as a prototype.

Self-Guiding & Documentation

A big advantage in my design is that I used components like a flight controller, which have a lot of documentation and tutorials available. This made it possible to actually get things working without starting from zero.

But at the same time I also ran into problems where documentation was missing or unclear, and then it becomes quite difficult to continue. I had situations where I had the hardware already, but I couldn’t properly use it because there was not enough information.

Entry Threshold

Getting started is not too hard if you already have some experience, but I would not say it is beginner friendly. You need to understand a bit of programming, electronics and also how to physically build things.

Especially combining everything together (like motors, structure and code) makes it more complex, because it is not just one system but multiple systems working together.

Interoperability

One thing that worked really well is that the components can be connected and combined quite easily. For example, the motors can be controlled through the flight controller and extended with other parts if needed.

This is important because rc systems are usually made from multiple parts working together, like servos, linkages and a controller, and that is exactly what I experienced while building this.

Novelty & Reflection

The idea itself is not completely new, since similar systems already exist and are widely used. But the process of building it myself still felt new, because I had to figure out many things on my own.

A big part of this project was solving problems when things did not work as expected, and sometimes making my own solutions when there was no ready-made option available.

Overall, I think this project shows quite well what tinkering is about: trying things, running into problems, and slowly improving the design until it actually works.
